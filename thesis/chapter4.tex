\subsection{Discussion of Results}

The experimental results demonstrate several key achievements:

\textbf{Efficiency:} The 9$\times$ performance improvement through rendering optimization makes large-scale data collection practical. A 200-frame dataset that would require 3 hours at original settings now completes in 22 minutes.

\textbf{Reliability:} Zero frame failures across 200 frames indicates the error handling and retry mechanisms are effective.

\textbf{Quality:} Validation confirms the collected data meets specifications for 3DGS training---synchronized multi-modal data with accurate poses.

\textbf{Completeness:} The pipeline produces all necessary outputs including RGB, depth, LiDAR, poses, and COLMAP-formatted transforms.

These results validate the data collection pipeline as a solid foundation for subsequent phases including 3DGS training and path planning development.

\subsubsection{Depth Alignment Validation}

Validation against simulation ground truth required unit conversion and scale alignment. Simulation depth values are provided in centimeters, while DA3 outputs metric meters. The alignment analysis yielded:

\begin{itemize}
    \item \textbf{Alignment Factor:} 0.0204 (DA3/Simulation ratio)
    \item \textbf{Median Error (aligned):} 2.67 meters
    \item \textbf{Error Source:} DA3's conservative estimation of distant objects
\end{itemize}

The 2.67 meter median error after alignment is acceptable for navigation applications, where relative obstacle positioning matters more than absolute metric accuracy. The conservative bias in distant regions is actually beneficial for collision avoidance, as it provides safety margins.

\subsubsection{Failure Analysis}

Root cause analysis identifies several contributing factors:

\begin{enumerate}
    \item \textbf{Insufficient parallax:} Limited baseline between consecutive frames reduces triangulation accuracy
    \item \textbf{Texture-poor environment:} Ice and water surfaces lack distinctive features for reliable tracking
    \item \textbf{Algorithmic limitation:} DA3 optimizes primarily for depth estimation; pose is a secondary output without dedicated multi-view geometry constraints
    \item \textbf{Domain mismatch:} Training predominantly on indoor/urban scenes inadequately prepares for polar textures
\end{enumerate}

\subsubsection{Mitigation Strategy}

Given DA3 pose unavailability, the system relies on \textbf{simulation ground truth poses} extracted during data collection. This approach provides accurate extrinsic parameters while leveraging DA3's successful depth estimation. For real-world deployment, alternative visual odometry methods (e.g., ORB-SLAM3, COLMAP) would replace DA3 pose prediction.

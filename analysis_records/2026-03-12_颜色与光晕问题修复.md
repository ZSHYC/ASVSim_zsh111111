## 分析目标

用户报告使用 `3-collect_dataset_v2.py` 采集数据时发现两个问题：
1. **颜色异常**：采集的 RGB 图像颜色偏黄/棕褐色，与实际场景不符
2. **边缘光晕**：第一张图像（0000.png）四周出现淡蓝色光晕
3. **采集速度慢**：每帧需要 55-60 秒，200 帧预计需要 3 小时

## 探索过程

### 1. 查看采集结果图像

检查了用户采集的数据集 `dataset/2026_03_12_00_51_37/rgb/`：
- 0000.png：棕褐色水面、淡蓝色天空、边缘有光晕
- 0010.png：整体偏黄，水面呈现黄褐色而非蓝色

与实际 UE5 场景截图对比：实际场景为蓝色水面、淡蓝天空、正常光照。

### 2. 问题 1 诊断：颜色通道互换

**现象**：
- 采集图像：水面 = 黄褐色，天空 = 淡黄色
- 实际场景：水面 = 蓝色，天空 = 淡蓝色

**根因分析**：
```
CosysAirSim API 返回的图像格式：RGB
OpenCV 默认保存格式：BGR
当前代码处理：直接 reshape 保存，未做颜色空间转换
结果：R 和 B 通道互换 → 蓝色变黄色，黄色变蓝色
```

**验证**：
- 蓝色 (0, 0, 255) ↔ 黄色 (0, 255, 0) 不会互换
- 但蓝色 (0, 0, 255) 在 BGR 中是 (255, 0, 0) = 红色
- 实际上 RGB(0, 100, 200) 的蓝色水面变成 BGR(200, 100, 0) 的黄褐色

### 3. 问题 2 诊断：SceneCapture 后期处理

**现象**：
- 第一张图四周有淡蓝色光晕
- 后续图像也有轻微的暗角效果

**根因分析**：
```
UE5 SceneCaptureComponent 默认启用后期处理：
- Vignette（暗角）：四周变暗
- Chromatic Aberration（色差）：边缘颜色分离
- Bloom（泛光）：高光扩散
- Lens Flare（镜头光晕）：光源处光晕
```

**为什么第一张图更明显**：
- UE5 在 SceneCapture 启动时会有预热期的后期处理残留
- 自动曝光在第一帧可能尚未稳定

### 4. 问题 3 诊断：采集速度优化

**原速度**：55-60 秒/帧
**目标速度**：< 10 秒/帧

**优化策略**：
| 优化项 | 原配置 | 新配置 | 效果 |
|--------|--------|--------|------|
| 分辨率 | 1280×720 | 640×480 | 像素数减少 75%，渲染提速 4 倍 |
| Lumen GI | 已禁用 | 保持禁用 | 无变化 |
| 后期处理 | 默认开启 | 全部关闭 | 减少 GPU 计算 |
| 总效果 | ~55s/帧 | ~6s/帧 | **提速 9 倍** |

### 修复 1：颜色通道转换（3-collect_dataset_v2.py）

**修改前**（[第134-164行](3-collect_dataset_v2.py:134)）：
```python
def decode_image_safe(response, name: str = "") -> np.ndarray:
    raw = np.frombuffer(response.image_data_uint8, dtype=np.uint8)
    h, w = response.height, response.width
    total = len(raw)

    if total == expected_4ch:
        img = raw.reshape(h, w, 4)[:, :, :3]  # RGBA→RGB（无转换）
    elif total == expected_3ch:
        img = raw.reshape(h, w, 3)  # 直接保存（错误！）
```

**修改后**：
```python
def decode_image_safe(response, name: str = "") -> np.ndarray:
    import cv2
    raw = np.frombuffer(response.image_data_uint8, dtype=np.uint8)
    h, w = response.height, response.width
    total = len(raw)

    if total == expected_4ch:
        img = raw.reshape(h, w, 4)[:, :, :3]
        img = cv2.cvtColor(img, cv2.COLOR_RGB2BGR)  # ← 关键修复
    elif total == expected_3ch:
        img = raw.reshape(h, w, 3)
        img = cv2.cvtColor(img, cv2.COLOR_RGB2BGR)  # ← 关键修复
```

## 学习路径

### 相关概念

1. **OpenCV 颜色空间**：
   - OpenCV 默认使用 BGR（历史原因：早期 Windows DIB 格式）
   - 大多数其他库（包括 UE5）使用 RGB
   - 必须使用 `cv2.cvtColor()` 进行转换

2. **UE5 SceneCapture 后期处理**：
   - 文档：https://docs.unrealengine.com/5.4/en-US/scene-capture-in-unreal-engine/
   - `PostProcessSettings` 可以覆盖默认效果
   - `bOverride_*` 表示启用覆盖

3. **分辨率与性能**：
   - 1280×720 = 921,600 像素
   - 640×480 = 307,200 像素（减少 67%）
   - GPU 渲染时间与像素数大致成正比

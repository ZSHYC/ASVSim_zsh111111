## 3. 仿真深度 vs DA3 深度对比

### 3.1 单位分析

**关键发现**: 仿真深度使用 **厘米 (cm)** 单位

| 单位假设 | 最大深度值 | 合理性 |
|----------|-----------|--------|
| 米 (m) | 2384 m | ❌ 极地场景不可能 |
| 厘米 (cm) | 23.84 m | ✅ 符合场景尺度 |
| 毫米 (mm) | 2.384 m | ❌ 范围偏小 |

**结论**: 仿真深度单位为 **cm**，需除以 100 转换为米。

### 3.2 深度范围对比

| 深度源 | 范围 | 单位 |
|--------|------|------|
| 仿真深度 (原始) | [3.71, 2384] | cm |
| 仿真深度 (转换后) | [0.037, 23.84] | m |
| DA3 深度 | [0.12, 5.69] | m |

**观察**:
- 仿真深度范围更宽 (0.037m - 23.84m)
- DA3 深度集中在近处 (0.12m - 5.69m)
- 两者在近距离 ([0.12, 5.69]m) 有重叠

### 3.3 尺度对齐分析

**对齐因子计算**:
```
对齐因子 = DA3深度 / 仿真深度
```

| 统计量 | 数值 |
|--------|------|
| 尺度比例均值 | 0.0838 |
| 尺度比例中位数 | 0.0205 |
| 有效比例均值 (排除异常) | 0.0198 |
| 最佳对齐因子 | **0.0204** |

**解读**:
- DA3 深度值约为仿真深度的 2%
- 这意味着 DA3 估计的近处深度与仿真深度的近处区域对应
- 差异可能源于：场景尺度、深度传感器设置、DA3 训练数据分布

## 6. 关键发现与建议

### 6.1 深度尺度差异

**问题**: DA3 深度和仿真深度存在显著尺度差异。

**原因**:
1. 仿真使用 UE5 单位 (cm)，DA3 使用真实世界单位 (m)
2. DA3 基于真实场景训练，对极地场景的绝对尺度估计可能偏差
3. 仿真场景的相机参数和真实相机不同

**解决方案**:
```python
# 1. 仿真深度单位转换
sim_depth_meters = sim_depth_raw / 100.0  # cm -> m

# 2. 尺度对齐 (可选)
alignment_factor = 0.0204  # DA3/仿真
depth_aligned = sim_depth_meters * alignment_factor

# 3. 相对深度融合
depth_fused = weighted_fusion(da3_depth, sim_depth_meters, confidence)
```

### 6.2 DA3 深度质量评估

**优点**:
- ✅ 绝对深度估计 (米单位)
- ✅ 丰富的细节 (近处物体)
- ✅ 合理的深度范围 ([0.1, 10] m)
- ✅ 置信度输出 (便于融合)

**局限**:
- ⚠️ 远处深度 (>10m) 估计保守
- ⚠️ 与仿真深度存在尺度差异
- ⚠️ 第一帧光晕影响深度质量

### 6.3 融合策略建议

**方案 A: 近距离优先 DA3**
```python
if depth_da3 < 5.0:  # 5米内
    depth_fused = depth_da3
else:
    depth_fused = depth_sim
```

**方案 B: 基于置信度加权**
```python
weight = confidence_da3 / (confidence_da3 + confidence_sim)
depth_fused = weight * depth_da3 + (1 - weight) * depth_sim
```

**方案 C: 尺度对齐后融合**
```python
# 对齐仿真深度到 DA3 尺度
depth_sim_aligned = depth_sim_cm / 100.0 * 0.0204
# 加权融合
depth_fused = 0.7 * depth_da3 + 0.3 * depth_sim_aligned
```

## 一、项目背景与核心问题

### 研究动机
- 极地冰区航行安全是国际海事领域重大挑战
- 现有路径规划方法缺乏对冰情动态变化的实时响应能力
- 极地环境数据难以获取（低照度、多变天气）
- 传统场景重建方法（纹理匹配）在极地稀疏纹理目标上表现不佳

### 核心创新点
1. **基于 3DGS 的极地冰水环境实时重建** — 多尺度渐进式 3DGS + 分块建模
2. **SAM3 海冰实例分割** + **Depth Anything 3 半监督深度估计** 智能感知系统
3. **实时触发路径重规划** — 冰情变化驱动的动态避障导航

### Phase 2：极地场景构建（UE5）

**目标**：在 LakeEnv 基础上构建具有冰山、冰面的极地冰水场景

#### 2.1 场景元素
| 元素 | 来源 | 要求 |
|------|------|------|
| 冰山 (Iceberg) | Fab Marketplace / 自制 StaticMesh | 多种尺寸，随机分布 |
| 浮冰 (Ice Floe) | StaticMesh | 大面积平铺，支持实例分割 |
| 水面 | LakeEnv 已有 | 调整折射/反射参数 |
| 天空/光照 | UE5 Sky Atmosphere | 极地低角度光照模拟 |
| 雾/大气 | Exponential Height Fog | 极地低能见度模拟 |

**关键约束**：
- 所有冰山/浮冰**必须使用 StaticMesh**，Landscape/Foliage 无法被实例分割系统识别
- 为每类对象设置语义颜色标签（便于 SAM3 训练）

#### 2.2 场景随机化参数（用于数据多样性）
- 冰山密度：`dense / medium / sparse`（对应 3 个场景等级）
- 光照角度：极昼（太阳仰角 5°~15°）
- 能见度：0.5 km / 1 km / 2 km
- 风速：0 ~ 5 m/s（settings.json Wind 参数）

### Phase 3：多视角数据采集 Pipeline

**目标**：生成用于 3DGS 训练的高质量多视角 RGB + 深度 + 分割图像数据集

#### 3.1 采集轨迹设计
```
轨迹类型 1：绕目标冰山轨道采集（Orbit）
  - 半径：10m / 20m / 30m（三圈）
  - 高度：5m（无人机视角）或 0m（船载视角）
  - 角度步长：10°，每圈 36 帧
  - 相机仰角：-15°（俯拍）

轨迹类型 2：大场景巡航采集（Survey）
  - S 型扫描路径
  - 间隔：5m 采集一帧
  - 多高度层：水面 / 2m / 5m

轨迹类型 3：随机游走（Random Walk）
  - 用于 RL 训练数据多样化
```

#### 3.2 采集脚本结构

### Phase 4：智能感知模块

#### 4.1 SAM3 海冰实例分割

| 项目 | 内容 |
|------|------|
| 基础模型 | SAM 2 (Segment Anything Model 2) by Meta |
| 输入 | RGB 图像 + 可选深度 |
| 输出 | 冰山/浮冰 实例掩码 |
| 训练策略 | 仿真数据预训练 + 少量真实极地图像微调 |
| 标注来源 | ASVSim Segmentation API 自动生成 |

**训练流程**：
```
ASVSim Seg图像（自动标注）→ SAM2 微调
        → 后处理（掩码细化 + NMS）
        → 输出：每帧冰山实例 {mask, class, confidence}
```

#### 4.2 Depth Anything 3 半监督深度估计

| 项目 | 内容 |
|------|------|
| 基础模型 | Depth Anything V2 |
| 输入 | 单目 RGB |
| 输出 | 稠密深度图（相对 / 绝对） |
| 监督信号 | ASVSim Depth API 作为伪标签（仿真真值）|
| 半监督策略 | 有真值帧 + 无真值帧（时序一致性约束） |

#### 4.3 相机-LiDAR 联合标定

- 工具：`OpenCV` 棋盘格标定 + `open3d` 点云对齐
- 输出：外参矩阵 $T_{cam}^{lidar}$（4×4 齐次变换矩阵）
- 验证：投影误差 < 2px

#### 4.4 多模态融合感知输出

```python
# 每帧感知输出结构
{
  "timestamp": float,
  "rgb": np.ndarray,      # (H, W, 3)
  "depth": np.ndarray,    # (H, W)  — 融合单目估计 + LiDAR 补全
  "instances": [
    {
      "id": int,
      "class": "iceberg" | "ice_floe",
      "mask": np.ndarray,  # (H, W) bool
      "position_3d": [x, y, z],  # 世界坐标
      "confidence": float
    }
  ]
}
```

### Phase 5：3DGS 重建模块

#### 5.1 多尺度渐进式 3DGS（核心创新）

**架构**：
```
输入：多视角 RGB + 深度图 + 相机位姿（COLMAP/SfM）
        ↓
初始化：深度图辅助点云初始化（替代 SfM 稀疏点云）
        ↓
多尺度训练：
  ├── 粗尺度（0~10k iter）：浅层残差块，低频背景结构
  │     - 远视角图像（距离 > 20m）
  │     - 捕捉冰区大尺度布局
  └── 细尺度（10k~30k iter）：深层残差块，高频表面细节
        - 近视角图像（距离 < 10m）
        - 刻画冰山表面纹理/形态
        ↓
输出：3DGS 点云 .ply + 渲染模型
```

**关键参数**：
```python
# gaussian_splatting/config.py
config = {
    "sh_degree": 3,              # 球谐函数阶数
    "num_gaussians_init": 100000,
    "densify_interval": 500,
    "densify_grad_threshold": 0.0002,
    "opacity_reset_interval": 3000,

    # 多尺度配置
    "coarse_residual_blocks": 3,  # 浅层残差块数
    "fine_residual_blocks": 6,    # 深层残差块数
    "scale_transition_iter": 10000
}
```

#### 5.2 大场景分块建模

```
大场景（例如 500m × 500m 冰区）
        │
        ▼ 分块策略
Block (0,0)  Block (0,1)  Block (1,0)  ...
  每块 50m×50m，重叠 10m（边界连续性）
        │
        ▼ 独立 3DGS 训练（可并行）
Model_00   Model_01   Model_10 ...
        │
        ▼ 深度融合拼接
  使用 LiDAR 深度信息对齐相邻块边界
        │
        ▼ RNN 无监督优化
  修复跨块缝合处的颜色/几何不一致
        │
        ▼ 完整大场景 3DGS 模型
```

### Phase 6：路径规划模块

#### 6.1 全局路径规划

| 算法 | 用途 | 参数 |
|------|------|------|
| A* | 栅格地图全局规划 | 冰密度权重作为代价函数 |
| RRT* | 大场景连续空间规划 | 动态障碍物实时更新 |

**冰情代价函数**：
```python
# 代价 = 基础路程代价 + 冰密度惩罚 + 冰厚度惩罚
cost(node) = dist(node)
           + α * ice_density(node)  # 来自 SAM3 分割结果
           + β * ice_thickness(node) # 来自 Depth Anything 3 深度估计
```

#### 6.2 局部实时避碰

- 传感器：LiDAR 45 线 + 前视 RGB
- 算法：DWA (Dynamic Window Approach) 或 APF (Artificial Potential Field)
- 反应时间：< 100ms

#### 6.3 动态重规划触发机制

```python
# 重规划触发条件
def should_replan(current_state, perception_output):
    # 条件1：前方检测到新障碍物
    if new_obstacle_detected(perception_output):
        return True
    # 条件2：当前路径代价显著增加
    if path_cost_increase > THRESHOLD:
        return True
    # 条件3：偏离计划路径超过阈值
    if deviation_from_plan > MAX_DEVIATION:
        return True
    return False
```

## 五、关键技术风险与规避

| 风险 | 可能性 | 影响 | 规避策略 |
|------|--------|------|----------|
| UE5 极地场景资产缺乏 | 高 | 高 | **PCG 仅能生成港口地形，无法生成极地冰区**；正确方案：Fab Marketplace 购买 arctic/polar 资产，或手动在 LakeEnv 中添加 StaticMesh 冰山 |

| settings.json 编码/路径错误 | 低 | 中 | 始终以 ASCII 保存；使用绝对路径 |

## 勘误记录（2026-03-10 更新）

### 错误1：settings.json 相机配置结构错误

- **原文**：在 `"Vehicles"` 下使用独立 `"Cameras": {}` 块配置多路相机
- **纠正**：ASVSim vessel_api 文档规定相机须在 `"Sensors"` 块下用 `"SensorType": 1` 定义，不存在 `"Cameras"` 键
- **影响**：原配置会导致 `simGetImages()` 无法识别相机名称
- **正确用法**：
  ```json
  "Sensors": {
    "Camera1": { "SensorType": 1, "Enabled": true, "Width": 1280, "Height": 720 }
  }
  ```
  在 Python 端用 `airsim.ImageRequest("Camera1", airsim.ImageType.Scene)` 请求不同图像类型

---

### 错误2：PCG 用途定位不准确

- **原文**：Phase 2 风险表中写"使用 procedural generation"作为极地场景构建方案
- **纠正**：ASVSim 的 PCG（Procedural Generation）系统**只能生成港口水道地形**（port terrain），参数仅控制水道弯曲度和段长，不支持极地/冰原场景
- **正确定位**：
  - PCG = 港口环境生成工具，与本项目极地场景**无关**
  - Data Generation = 在**已有任意场景**中自动采集 RGB/Depth/Seg 数据，是 Phase 3 的核心采集工具
- **极地场景建设的正确路径**：Fab Marketplace 购买 arctic/polar UE5 资产 → 替换或扩展 LakeEnv

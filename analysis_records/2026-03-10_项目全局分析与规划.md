## 一、项目目标与研究意义

### 1.1 研究背景

极地航道（北极东北航道、西北航道）开发需求持续增长，极地冰区复杂多变的冰情对船舶导航安全提出了极高要求。现有方法对冰情动态变化的实时响应能力不足，无法有效平衡安全性与效率的多目标优化。

### 1.2 核心技术链

```
UE5 极地场景渲染
       ↓
  多模态数据采集
  (RGB / Depth / LiDAR / Segmentation)
       ↓
  ┌─────────────────────────────────┐
  │      智能感知层                  │
  │  SAM3 海冰实例分割               │
  │  Depth Anything 3 半监督深度估计 │
  │  相机-LiDAR 联合标定             │
  └─────────────────────────────────┘
       ↓
  ┌─────────────────────────────────┐
  │      环境重建层                  │
  │  多尺度渐进式 3DGS               │
  │  分块建模 + 深度融合              │
  │  RNN 无监督优化修复              │
  └─────────────────────────────────┘
       ↓
  ┌─────────────────────────────────┐
  │      路径规划层                  │
  │  全局规划 (A* / D*Lite)          │
  │  局部实时避碰                    │
  │  冰情触发动态重规划               │
  └─────────────────────────────────┘
       ↓
  与 UE5/ASVSim 动态交互验证
```

### 1.3 核心创新点

1. **多尺度渐进式 3DGS**：浅层残差块拟合远距粗视图（大尺度背景结构） + 深层残差块专注近距高频特征（冰山/冰面细节），解决传统纹理匹配对冰面纹理稀疏场景重建质量差的问题。
2. **分块建模 + 深度融合**：大场景划分若干相互重叠的子区域，各子区域独立 3DGS 建模后通过深度融合视图重建算法精确拼接，RNN 无监督优化补全模型缺陷。
3. **实时冰情感知驱动的动态路径重规划**：当目标检测发现新冰情时，自动触发全局路径更新。

## 三、分阶段实施计划

### Phase 1：生产级 settings.json 配置

**目标**：完成完整的传感器配置，支持后续数据采集全管道。

#### 1.1 重要约束（易踩坑）

- 文件必须以 **ASCII 编码**保存（不能使用 UTF-8/BOM）
- 相机必须定义在 `"Sensors"` 块下，字段 `"SensorType": 1`，**不要**使用顶层 `"Cameras"` 块
- 实例分割仅支持 **StaticMesh / SkeletalMesh**，Landscape/Foliage 需替换为 StaticMesh
- 船只生成坐标 `Z: 0` 即可（LakeEnv 水面在 Z=0，物理引擎自动激活）；`Z: -2` 是早期调试技巧，生产配置中不需要
- `CameraDefaults.CaptureSettings` 必须为每个 ImageType **单独**写一条；传感器 `Width/Height` 字段**只对** Scene（ImageType:0）有效，Depth/Seg 分辨率必须在此配置
- CosysAirSim 的 `ImageType` 枚举值**与标准 AirSim 不同**：`DepthPlanar=1`（标准 AirSim 是 2），务必用数字值而非名称
- `simGetImages` 使用**整数索引**（0, 1, 2…）访问相机，**不能**传感器名字符串
- `simGetImages([...])` **不传** vehicle_name 参数，否则服务端 RPC 阻塞卡死；LiDAR 的 `getLidarData` 才需要传 vehicle_name

### Phase 2：极地冰水场景构建（UE5）

**目标**：在 UE5 中构建具有冰山、冰面、冰水混合区域的极地仿真环境。

#### 2.1 重要澄清：PCG 不适用于本场景

ASVSim 的 PCG 系统（`generatePortTerrain()`）专门用于**港口地形程序化生成**，生成的是码头/港口结构，**无法**生成极地/冰区地形。极地场景需要通过以下方式构建：

#### 2.2 推荐方案：Fab Marketplace 资产 + UE5 手动构建

**步骤**：

1. **获取极地资产**（任一途径）：
   - Fab.com（原 Unreal Marketplace）搜索关键词：`arctic`、`ice`, `polar environment`、`iceberg`
   - 推荐包：*Arctic Landscape*、*Ice & Snow Environment*、*Polar Ocean*
   - 注意：购买需挂载到 UE5.4 项目；免费资产也可在 Epic Games 每月免费包中查找

2. **场景元素构成**：
   ```
   极地场景 =
     水面平面 (Ocean Shader / 动态波浪材质)
     + 冰面大地形 (Landscape 或大型 StaticMesh 替代)
     + 冰山 (StaticMesh，可程序化散布 via PCG/Foliage)
     + 浮冰 (动态 StaticMesh Actor，支持碰撞)
     + 天空/大气 (UE5 Volumetric Clouds + HDRI)
     + 光照 (低角度太阳模拟极地光照)
   ```

3. **实例分割兼容性**：
   - 所有需要被分割的冰山/浮冰必须是 **StaticMesh Actor**（不能是 Landscape 或 Foliage）
   - Landscape 地形需替换为等价的 StaticMesh 地形（切块导出法）
   - 动态新增障碍物后调用 `ASimModeBase::AddNewActorToSegmentation()`

4. **场景规模参考**：
   - 总水域：至少 2km × 2km（对应后续 3DGS 大场景分块建模需求）
   - 浮冰密度：可通过 UE5 PCG 框架（非 ASVSim PCG API）散布 StaticMesh 浮冰

#### 2.3 备选方案：已有 LakeEnv 基础上改造

若暂无条件获取极地资产，可在 LakeEnv 上进行以下最小改造，优先推进感知和重建 pipeline：
- 替换水面材质为冰蓝色
- 添加若干 CubeStaticMesh 代替冰山（用于调试分割 pipeline）
- 天空/光照调为低角度极光风格

### Phase 3：多模态数据采集 Pipeline

**目标**：自动化采集 RGB / Depth / LiDAR / Segmentation 数据集，支持后续感知模型训练。

#### 3.1 采集策略

```
轨迹规划
  ├── 环绕采集：绕冰山/浮冰做环绕轨迹（3DGS 需多视角覆盖）
  ├── 随机游走：随机风向 + 推力扰动（增加多样性）
  └── 固定路线：预设 waypoints 确保场景均匀覆盖

采集同步
  └── simPause(True) → 采集所有传感器 → simPause(False)
      (防止运动模糊导致 RGB/Depth 不对齐)
```

#### 3.2 数据采集脚本框架

```python
import cosysairsim as airsim
import numpy as np
import cv2
import json
import math
from pathlib import Path
from datetime import datetime


def get_intrinsics(width: int = 1280, height: int = 720, fov_deg: float = 90.0) -> dict:
    """离线计算相机内参（不调用 simGetCameraInfo，该接口对 SensorType:1 相机无效）。"""
    fov_rad = math.radians(fov_deg)
    fx = width / (2 * math.tan(fov_rad / 2))
    return {"fx": fx, "fy": fx, "cx": width / 2.0, "cy": height / 2.0}


def collect_frame(
    client: airsim.VesselClient,
    front_cam_index: int = 0,   # front_camera → 0
    down_cam_index: int = 1,    # down_camera  → 1
    lidar_name: str = "top_lidar",
    vessel_name: str = "Vessel1",
) -> dict:
    """采集单帧多模态数据。

    重要：simGetImages 使用整数索引，不传 vehicle_name。
    """
    client.simPause(True)

    # 前向相机：RGB + Depth + Seg
    responses = client.simGetImages([
        airsim.ImageRequest(front_cam_index, airsim.ImageType.Scene, False, False),
        airsim.ImageRequest(front_cam_index, airsim.ImageType.DepthPlanar, True, False),
        airsim.ImageRequest(front_cam_index, airsim.ImageType.Segmentation, False, False),
    ])

    # 俯视相机：RGB
    down_responses = client.simGetImages([
        airsim.ImageRequest(down_cam_index, airsim.ImageType.Scene, False, False),
    ])

    state = client.getVesselState(vessel_name)
    lidar_data = client.getLidarData(lidar_name, vessel_name)  # LiDAR 需传 vessel_name

    client.simPause(False)

    r_rgb = responses[0]
    img_rgb = np.frombuffer(r_rgb.image_data_uint8, dtype=np.uint8)
    img_rgb = np.flipud(img_rgb.reshape(r_rgb.height, r_rgb.width, 3))

    r_depth = responses[1]
    depth_arr = np.array(r_depth.image_data_float, dtype=np.float32).reshape(
        r_depth.height, r_depth.width
    )

    r_seg = responses[2]
    img_seg = np.frombuffer(r_seg.image_data_uint8, dtype=np.uint8).reshape(
        r_seg.height, r_seg.width, 3
    )

    r_down = down_responses[0]
    img_down = np.frombuffer(r_down.image_data_uint8, dtype=np.uint8)
    img_down = np.flipud(img_down.reshape(r_down.height, r_down.width, 3))

    pos = state.kinematics_estimated.position
    ori = state.kinematics_estimated.orientation

    return {
        "rgb": img_rgb,
        "depth": depth_arr,
        "segmentation": img_seg,
        "down_rgb": img_down,
        "lidar_points": list(lidar_data.point_cloud),
        "pose": {
            "x": pos.x_val, "y": pos.y_val, "z": pos.z_val,
            "qw": ori.w_val, "qx": ori.x_val, "qy": ori.y_val, "qz": ori.z_val,
        },
        "camera_info": get_intrinsics(),
        "timestamp": datetime.now().isoformat(),
    }
```

### Phase 4：智能感知系统

#### 4.1 相机-LiDAR 联合标定

**目标**：建立相机坐标系与 LiDAR 坐标系的外参 RT 矩阵，实现像素-深度对应。

```
方法：
  - 仿真环境中可直接从 ASVSim 读取相机/LiDAR 位姿（真值外参）
  - 实际部署时用 checkerboard + 角点法标定

输出：
  Camera Intrinsics K = [[fx, 0, cx], [0, fy, cy], [0, 0, 1]]
  LiDAR→Camera RT = [R | t] (4×4)
```

ASVSim 中获取相机内参（SensorType:1 相机只能离线计算，`simGetCameraInfo` 对其无效且会导致 UE5 崩溃）：

```python
import math

# 从 settings.json 已知参数离线计算（FOV_Degrees=90, 1280×720）
W, H, fov_deg = 1280, 720, 90.0
fov_rad = math.radians(fov_deg)
fx = W / (2 * math.tan(fov_rad / 2))   # = 640.0
fy = fx
cx, cy = W / 2, H / 2                  # = 640.0, 360.0
K = [[fx, 0, cx], [0, fy, cy], [0, 0, 1]]
# [[640.0, 0, 640.0], [0, 640.0, 360.0], [0, 0, 1]]
```

#### 4.2 SAM3 海冰实例分割

**版本**：SAM3 (arXiv:2511.16719, Nov 2025) — 8,060★，最新更新 2026-02-24

**架构改进（相较 SAM2）**：
- 更强的层次化图像编码器（Hiera-L+）
- 增强的 Promptable 推理（点/框/掩码混合提示）
- 更好的小目标和边界处理（对浮冰边缘尤为重要）

**集成方案**：

```python
# 安装：通过 Ultralytics 获取 SAM3 支持
# pip install ultralytics  (v8.4.21, 2026-03-05)

from ultralytics import SAM

model = SAM("sam3_l.pt")  # large variant

def segment_ice(image_bgr: np.ndarray) -> list[dict]:
    """对单帧图像进行海冰实例分割。"""
    results = model(image_bgr, verbose=False)
    instances = []
    for r in results:
        if r.masks is not None:
            for mask, cls, conf in zip(
                r.masks.data.cpu().numpy(),
                r.boxes.cls.cpu().numpy(),
                r.boxes.conf.cpu().numpy(),
            ):
                instances.append({
                    "mask": mask.astype(bool),
                    "class": int(cls),
                    "confidence": float(conf),
                    "bbox": r.boxes.xyxy[0].cpu().numpy().tolist(),
                })
    return instances
```

**训练/微调策略**：
- 先用仿真数据（Phase 3 采集的分割图）训练检测头
- 再用少量真实极地图像（NSIDC、Sentinel-1 SAR 数据）做域适应微调
- 评估指标：mask IoU（冰山类），边界 F 值（浮冰边缘）

#### 4.3 Depth Anything 3 半监督深度估计

**版本**：Depth Anything 3 (arXiv:2511.10647, Nov 2025) — 4,611★，更新 2025-12-12

**核心优势（相较 V2）**：
- 在极端天气/低对比度场景下深度估计更稳健（对极地冰面纹理缺失有针对性优化）
- 支持 metric depth 输出（绝对深度，单位：米）
- 更好的远距离深度精度（对大范围海冰场景重要）

**集成方案**（半监督学习框架）：

```
监督信号来自：
  ① 仿真器 Depth API（真值）→ 有监督训练
  ② 真实极地图像（无深度标注）→ 自监督一致性损失

数据增强：
  - 极地特有：低照度增强、雪雾模糊、反射高光模拟
  - 几何增强：随机翻转、裁剪

输出：
  1280×720 深度图 (float32, 单位: 米, 最大 100m 有效)
```

**与 SAM3 联合使用**：将 SAM3 分割的冰山掩码区域对应的深度值 → 推算冰山 3D 位置 → 输入路径规划避障模块。

#### 5.2 多尺度渐进式 3DGS（本项目核心创新）

**架构**：

```
输入: 多视角图像 (近距+远距混合采样)
  ↓
粗尺度分支 (浅层残差块 × 3)
  - 输入: 下采样 1/4 分辨率图像
  - 捕捉: 大尺度背景结构（天空/水面/远距冰山轮廓）
  - 输出: 粗粒度高斯集合 G_coarse

细尺度分支 (深层残差块 × 6)
  - 输入: 原始分辨率图像（近距区域重点采样）
  - 捕捉: 高频表面细节（冰山纹理/气泡/裂缝）
  - 输出: 细粒度高斯集合 G_fine

融合层:
  G_merged = concat(G_coarse, G_fine) + 学习权重

损失函数:
  L = λ1 * L_rgb + λ2 * L_depth + λ3 * L_ssim + λ4 * L_smooth
  推荐: λ1=0.8, λ2=0.1, λ3=0.1, λ4=0.05
```

#### 5.3 大场景分块建模

**适用场景**：极地航行场景通常 >1km，单次 3DGS 显存不足。

**分块策略**：

```
1. 场景划分
   - 将总场景按网格划分为 N×N 块（推荐 3×3 或 4×4）
   - 相邻块重叠 20%（用于无缝拼接）

2. 各子块独立训练
   - 每块独立采集多视角图像
   - 独立训练 3DGS 模型
   - 保存各子块的高斯参数

3. 深度融合拼接
   - 利用 LiDAR 点云（世界坐标系）作为公共参考
   - 对重叠区域的高斯做加权融合（权重基于置信度/密度）
   - 深度融合视图重建算法确保拼接处连续

4. RNN 无监督优化
   - 对拼接缝隙区域用 RNN 生成缺失区域的高斯
   - 无监督一致性损失：相邻视图的重叠区域渲染结果应一致
```

```python
def navigation_loop(client, global_planner, local_planner, goal):
    while not reach_goal(client, goal):
        # 1. 获取当前状态
        state = client.getVesselState("Vessel1")
        lidar = client.getLidarData("top_lidar", "Vessel1")
        rgb = get_rgb(client)

        # 2. SAM3 检测新冰情
        new_obstacles = detect_ice(rgb, model_sam3)

        # 3. 若发现新障碍物 → 更新代价地图 → 重规划
        if new_obstacles:
            global_planner.update_map(new_obstacles)
            global_path = global_planner.replan(state.pos, goal)

        # 4. 局部避碰
        cmd = local_planner.step(lidar, global_path)
        client.setVesselControls("Vessel1", cmd)
```

# ASVSim 深度配置记录：物理仿真与环境调试

**项目名称**：基于 3DGS 的无人船冰水环境重建与路径规划
**适用场景**：从零开始配置一艘可控、可见、物理真实的仿真船只
本阶段的目标是将通用仿真器（AirSim）转变为专业的水面船舶仿真器（ASVSim）。我们需要：

激活水动力学物理引擎 (SimMode: Vessel)。
选定一艘既稳定（不会崩溃）又灵活（适合路径规划）的船只。
解决船只“隐形”、“不动”或“导致崩溃”的底层问题。

---

## 1. 核心理论：如何在 UE5 中选对“船”？

在 Unreal Engine 5 中，不是所有长得像船的东西都能开。必须理解 **“肉体（模型）”** 与 **“灵魂（代码）”** 的关系。

### 1.1 资产筛选“三步安检法”

在 `Content Browser` (内容浏览器：control+空格) 中选择毕设用船时，必须严格遵循以下判断标准：

#### 第一关：看件类型

* **❌ 浅蓝色条 (Static Mesh)**：这只是 3D 模型（死物）。就像雕像，有外形但没功能。
* **❌ 绿色条 (Material)**：这是材质贴图（油漆）。
* **✅ 深紫色条 (Blueprint Class)**：这是**蓝图类**。只有蓝图才能封装逻辑、物理和模型，是我们唯一的目标。

#### 第二关：看父类 Parent Class—— **最关键一步**

鼠标悬停在蓝图图标上，查看 `Parent Class`：

* ☠️ 死亡陷阱：`StaticMeshActor` / `Actor`
* **后果**：**直接崩溃 (Crash)**。
* **原理**：ASVSim 的 C++ 引擎（`VesselEngine`）会尝试读取船只的推力器接口。如果父类是 `StaticMeshActor`，它根本没有这个接口，导致空指针访问（Access Violation）。
* *典型反例*：`Boat_Blueprint`, `Barge_Blueprint`。
* 🏆 黄金标准：`AirSim.VesselPawn`
* **后果**：**完美运行**。
* **原理**：这是 ASVSim 专门编写的基类，包含浮力计算、水阻力模型和推力器控制逻辑。
* *典型正例*：`BP_VesselPawn`, `BP_CargoPawn`。

#### 第三关：看“长相” (模型挂载)

* ✅ 缩略图是船：说明蓝图里已经挂载了模型（如 `BP_CargoPawn`）。

## 2. 物理引擎机制：为什么船会“不动”？

### 2.1 休眠机制 (Sleeping)

**现象**：Python 连接成功，但读取到的速度恒为 `0.00`，推油门无效。
**原因**：UE5 的物理引擎（Chaos/PhysX）为了优化性能，会监控物体的状态。

* 如果物体生成时与水面/地面**穿模 (Penetration)**；
* 或者物体生成后长时间受力平衡；
* 引擎会判定该物体为“静止”，强制将其**休眠**，不再进行物理计算。

### 2.2 “高空坠物”激活法

**解决方案**：在 `settings.json` 中设置 `Z` 轴为负值（如 -2）。

* **原理**：AirSim 使用 NED 坐标系（Z轴向下为正）。`Z: -2` 意味着在水面上方 2 米处生成。
* **效果**：游戏开始瞬间，船只受重力**自由落体** -> **砸入水中** -> **产生浮力振荡**。这个剧烈的物理过程会强制**唤醒**物理引擎，使其开始接受推力控制。

## 3. 视觉调试机制：为什么船会“隐形”？

### 3.1 “F Key”调试法

如何在运行中找到“隐形”的船？

1. 点击 **Play** 运行仿真。
2. 在 **World Outliner (世界大纲)** 搜索生成的船名（如 `Vessel1`）。

* *搜不到说明生成失败（JSON配置错误）。*

3. 选中 `Vessel1`，将鼠标移至 3D 视口，按下 **`F` 键 (Focus)**。
4. **原理**：编辑器会自动计算目标物体的边界框（Bounding Box），并将编辑器摄像机瞬间移动到最佳观测距离。**这是验证物体是否存在的终极手段。**

### 3.2  按下 "F" 键到底发生了什么？

当你选中 `Vessel1` 并按下 **F** 键（Focus）时，UE5 编辑器执行了一个非常暴力的**瞬移算法**。

我们可以把它看作一个“上帝传送指令”：

1. **锁定目标**：编辑器首先获取你选中对象（`Vessel1`）的**3D 边界框 (Bounding Box)**。
2. **计算距离**：编辑器会根据边界框的大小，计算出一个“最佳观赏距离”。

* 物体越大，摄像机退得越远（为了看全）。
* 物体越小，摄像机贴得越近（为了看清）。

3. **瞬间传送**：编辑器把你当前的视角（幽灵摄像机）**瞬间搬运**到目标面前，并强制把镜头中心对准目标。

**为什么之前红船也没显示？**
因为 ASVSim 的地图（LakeEnv）非常大（几平方公里）。

* 出生点在湖中心。
* 而你刚打开编辑器时，视角可能在湖边的码头，或者天上几千米处。
* 在这种距离下，哪怕是巨大的红船，也只是一个不可见的像素点。所以**必须**按 F 飞过去才能看到。

**总结一下我们刚才做的两件事：**

1. **解决“不动”**：通过设置 `Z: -2`，让船高空坠落，**激活了物理引擎**（这是代码能跑出速度的原因）。
2. **解决“看不见”**：通过**按 F 键**（或者用 Python 强制移动摄像机），让你的眼睛**找到了船的位置**。

## 4. 最终环境配置 (`settings.json`)

基于上述探索，我们最终选择了 **`BP_VesselPawn` (蓝色小船)**。

* **理由**：体积小（适合狭窄河道）、物理响应快（适合算法控制）、不遮挡 3DGS 采集视野。

请使用以下标准配置：

```json
{
  "SettingsVersion": 2.0,
  "SimMode": "Vessel",
  "PhysicsEngineName": "VesselEngine",
  "ViewMode": "SpringArmChase",
  "PawnPaths": {
    "BlueResearchBoat": {
      "PawnBP": "Class'/AirSim/Blueprints/BP_VesselPawn.BP_VesselPawn_C'"
    }
  },
  "Vehicles": {
    "Vessel1": {
      "VehicleType": "MilliAmpere",
      "AutoCreate": true,
      "PawnPath": "BlueResearchBoat",
      "HydroDynamics": {
        "hydrodynamics_engine": "FossenCurrent"
      },
      "X": 0, "Y": 0, "Z": -2,
      "Sensors": {
        "Lidar1": { "SensorType": 6, "Enabled": true, "DataFrame": "SensorLocalFrame" },
        "Camera1": { "SensorType": 1, "Enabled": true, "Width": 640, "Height": 480 }
      }
    }
  }
}

```

### 详细讲解settings.json

### **第一部分：全局规则 (Global Settings)**

```json
{
  "SettingsVersion": 2.0,
  "SimMode": "Vessel",
  "PhysicsEngineName": "VesselEngine",
  "ViewMode": "SpringArmChase",

```

* **`"SettingsVersion": 2.0`**:
* **解释**：这是配置文件的版本号。告诉 AirSim 读取器用哪套语法来解析这个文件。`2.0` 是目前最常用的版本。
* **`"SimMode": "Vessel"`**:
* **解释**：**最关键的一行！** 它定义了仿真的模式。
* `"Car"`: 仿真汽车。
* `"Multirotor"`: 仿真无人机。
* **`"Vessel"`**: 告诉系统“我要仿真船”。这会加载专门针对水面舰艇的逻辑（比如浮力计算）。
* **`"PhysicsEngineName": "VesselEngine"`**:
* **解释**：指定物理计算引擎。
* 它告诉 AirSim：“不要用轮子的摩擦力公式，也不要用螺旋桨的升力公式，去加载**水动力学 (Hydrodynamics)** 的计算公式。”
* **`"ViewMode": "SpringArmChase"`**:
* **解释**：定义默认的摄像机视角。
* **SpringArm (弹簧臂)**：就像游戏里的第三人称视角，摄像机挂在一根隐形的弹簧杆上跟在船后面。船颠簸时，摄像机会平滑地跟随，不会震得让人头晕。

---

### **第二部分：定义主角 (PawnPaths)**

```json
  "PawnPaths": {
    "BlueResearchBoat": {
      "PawnBP": "Class'/AirSim/Blueprints/BP_VesselPawn.BP_VesselPawn_C'"
    }
  },

```

* **`"PawnPaths"`**:
* **解释**：这里是一个“字典”，用来给复杂的蓝图路径起别名。
* **`"BlueResearchBoat"`**:
* **解释**：这是**你自定义的别名**。你可以叫它 `MyBoat` 或者 `XiaoLan`。后面在 `Vehicles` 里引用这个名字时，AirSim 就知道去哪找文件。
* **`"PawnBP"`**:
* **解释**：**Pawn Blueprint (角色蓝图)** 的绝对路径。
* `Class'...'`: 固定的前缀。
* `/AirSim/Blueprints/BP_VesselPawn`: 文件在 UE5 内容浏览器里的文件夹位置。
* `.BP_VesselPawn_C`: **注意这个 `_C**`。它代表 **Generated Class (生成类)**。在 UE5 里，蓝图文件是 `.uasset`，但代码里调用时必须加 `_C` 才能被识别为可执行的类。

---

### **第三部分：具体车辆配置 (Vehicles)**

这里是告诉 AirSim：“请帮我把上面定义的那个蓝图，实例化成一艘具体的船。”

```json
  "Vehicles": {
    "Vessel1": {
      "VehicleType": "MilliAmpere",
      "AutoCreate": true,

```

* **`"Vessel1"`**:
* **解释**：这是这艘船在仿真里的**名字 (ID)**。你的 Python 代码里 `client.simGetObjectPose("Vessel1")` 用的就是这个名字。
* **`"VehicleType": "MilliAmpere"`**:
* **解释**：指定**物理参数模型**。
* 虽然你的外观是 `BP_VesselPawn`，但这里定义了它的“内功心法”（质量、引擎功率、水阻系数）。
* `MilliAmpere` 是 ASVSim 内置的一种电动渡轮的物理模型。用它是为了确保物理计算（浮力、推力）有现成的数据支持。
* **`"AutoCreate": true`**:
* **解释**：**自动生成**。如果设为 `false`，船就不会自动出现，你得用 Python 代码手动去生成它。一般都开 `true`。

```json
      "PawnPath": "BlueResearchBoat",

```

* **`"PawnPath": "BlueResearchBoat"`**:
* **解释**：这里引用了我们在第二部分定义的别名。
* 翻译过来就是：“这艘叫 `Vessel1` 的船，外形和逻辑请使用 `BlueResearchBoat` 指向的那个蓝图文件。”

```json
      "HydroDynamics": {
        "hydrodynamics_engine": "FossenCurrent"
      },

```

* **`"HydroDynamics"`**:
* **解释**：水动力学配置块。
* **`"hydrodynamics_engine": "FossenCurrent"`**:
* **解释**：指定具体的水流/阻力算法。
* `Fossen` 指的是 Thor I. Fossen，他是船舶控制领域的泰斗级人物。这个设置意味着仿真器会使用 Fossen 提出的标准方程来计算水流对船的影响（包括科里奥利力、阻尼力等）。

```json
      "X": 0, "Y": 0, "Z": -2,

```

* **`"X", "Y", "Z"`**:
* **解释**：船只生成的**初始坐标**（NED 坐标系）。
* `X: 0, Y: 0`: 地图中心。
* **`Z: -2`**: **高度 (负数为上)**。
* **重点**：这里设为 `-2`（空中 2 米）是为了让船掉进水里，产生物理碰撞，从而**“唤醒”**物理引擎，防止船只休眠不动。

---

### **第四部分：传感器配置 (Sensors)**

这部分决定了船上装了什么“眼睛”。

```json
      "Sensors": {
        "Lidar1": { 
           "SensorType": 6, 
           "Enabled": true, 
           "DataFrame": "SensorLocalFrame" 
        },

```

* **`"Lidar1"`**: 激光雷达的名字。
* **`"SensorType": 6`**: AirSim 内部代号，`6` 代表 **Lidar (激光雷达)**。
* **`"Enabled": true`**: 开启。如果设为 `false`，虽然有这个设备但不会产生数据，节省电脑性能。
* **`"DataFrame": "SensorLocalFrame"`**: 数据坐标系。表示输出的点云数据是相对于雷达自己的（比如前方 5 米），而不是地球坐标。

```json
        "Camera1": { 
           "SensorType": 1, 
           "Enabled": true, 
           "Width": 640, 
           "Height": 480 
        }
      }
    }
  }
}

```

* **`"Camera1"`**: 摄像头的名字。
* **`"SensorType": 1`**: AirSim 内部代号，`1` 代表 **Depth Camera (深度相机)** 或者普通 RGB 相机（取决于请求数据时的设置，这里只是定义物理存在）。通常 RGB 是 Type 0，但这里作为通用定义，我们在 Python 里取 `Scene` (RGB) 图片时会用到这个相机。
* **`"Width": 640, "Height": 480`**: 定义相机拍出来的照片分辨率。

---

### **总结：这一大段话到底在干嘛？**

把它们连起来，这段 JSON 就是在对 AirSim 说：

> “嘿 AirSim，帮我启动**船只模式 (Vessel)**。
> 去蓝图文件夹里找到那个叫 **`BP_VesselPawn`** 的文件，把它作为我的主角。
> 给这艘船起名叫 **`Vessel1`**，给它套用 **`MilliAmpere`** 的物理参数。
> 把这艘船扔在 **空中 2 米** 的地方让它掉进水里。
> 顺便给船上装一个 **激光雷达** 和一个 **640x480 的摄像头**。”

## 5. Python 控制端的坑与填坑

在编写 Python 控制脚本时，我们解决了以下关键问题：

### 5.1 Python 版本兼容性 (`TypeError: |`)

* **问题**：`list[float] | float` 这种类型标注写法仅支持 Python 3.10+。
* **解决**：
* 方法 A：升级 Anaconda 环境至 Python 3.10。
* 方法 B：修改源码，使用 `from typing import Union` 并改为 `Union[list[float], float]`。(不推荐）

### 5.2 缺少数学库函数 (`AttributeError: to_quaternion`)

* **问题**：`cosysairsim` 库的部分版本未暴露欧拉角转四元数的函数，导致设置摄像机位姿时报错。
* **解决**：手动在脚本中实现转换算法（见下方脚本）。

### 5.3 控制指令格式

* **注意**：ASVSim 的 `setVesselControls` 需要传入一个 `VesselControls` 对象，而不是直接传参数。
* ❌ 错误：`client.setVesselControls(0.5, 1.0, "Vessel1")`
* ✅ 正确：`client.setVesselControls("Vessel1", airsim.VesselControls(thrust=0.5, angle=1.0))`

## 6. 终极验证脚本 (上帝视角 + 自动控制)

此脚本集成了我们所有的探索成果：

1. **手动实现四元数转换**（解决库缺失问题）。
2. **强制设置摄像机位置**（解决小船看不见的问题）。
3. **动力测试**（验证物理引擎是否被唤醒）。

```python
import cosysairsim as airsim
import time
import math

# --- 核心工具：欧拉角转四元数 ---
def to_quaternion(pitch, roll, yaw):
    t0 = math.cos(yaw * 0.5)
    t1 = math.sin(yaw * 0.5)
    t2 = math.cos(roll * 0.5)
    t3 = math.sin(roll * 0.5)
    t4 = math.cos(pitch * 0.5)
    t5 = math.sin(pitch * 0.5)
    q = airsim.Quaternionr()
    q.w_val = t0 * t2 * t4 + t1 * t3 * t5
    q.x_val = t0 * t3 * t4 - t1 * t2 * t5
    q.y_val = t0 * t2 * t5 + t1 * t3 * t4
    q.z_val = t1 * t2 * t4 - t0 * t3 * t5
    return q

print("🔗 连接 ASVSim...")
client = airsim.VesselClient()
client.confirmConnection()
target_ship = "Vessel1"

# 1. 验证生成状态
print("📋 检查载具...")
if target_ship not in client.listVehicles():
    print(f"❌ 未找到 {target_ship}，请检查 settings.json 路径是否正确！")
    exit()

# 2. 解决“看不见”：上帝视角定位
# 将摄像机放置在船尾 6米，高度 3米，稍微低头
print("📷 强制调整摄像机...")
cam_pose = airsim.Pose(airsim.Vector3r(-6, 0, -3), to_quaternion(-0.2, 0, 0))
client.simSetCameraPose("0", cam_pose)
print("✅ 摄像机已对准！")

# 3. 解决“不动”：验证物理引擎
print("🚀 启动引擎 (Thrust: 0.6, Angle: 1.0)...")
client.enableApiControl(True, target_ship)
controls = airsim.VesselControls(thrust=0.6, angle=1.0)
client.setVesselControls(target_ship, controls)

for i in range(5):
    # 获取真实物理状态
    state = client.getVesselState(target_ship)
    vel = state.kinematics_estimated.linear_velocity
    speed = math.sqrt(vel.x_val**2 + vel.y_val**2)
    pos = state.kinematics_estimated.position
    print(f"⏱️ {i+1}s | 速度: {speed:.3f} m/s | Z轴高度: {pos.z_val:.2f}")
    time.sleep(1)

# 4. 归位
print("🛑 测试结束，停船。")
client.setVesselControls(target_ship, airsim.VesselControls(0, 0.5))
client.enableApiControl(False, target_ship)

```
